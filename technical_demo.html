Processing Pipeline
PDF In → Validated Data Out

Four sequential stages transform unstructured contractor and inspector PDFs into reconciled, validated, audit-ready records. Each stage is independent and configurable.

1

Document Ingestion

DoclingPyMuPDF fallbackPDF → Markdown

Accepts any PDF format — scanned forms, typed reports, multi-page tables. Docling preserves table structures critical for quantity extraction. If Docling fails on complex layouts, automatic fallback to PyMuPDF ensures zero total failures.

2

AI Extraction

Ollama + Llama 3.2Local LLMStructured JSON

Local LLM processes markdown and extracts structured fields: labor hours, equipment counts, material quantities, dates, and change order references. Pydantic schema enforcement means the LLM output matches expected types or gets retried (up to 3 attempts).

3

Reconciliation & Validation

Pydantic V24-Layer ValidationVariance Detection

Side-by-side comparison of contractor vs. inspector data. Flags discrepancies exceeding 5% threshold. Four validation layers (type → range → logic → human review) boost raw 85% accuracy to 95%+. Critical variances escalated for Contract Administrator review.

4

Storage & Audit Trail

SQLiteParameterized QueriesACID Compliant

Every extraction, validation result, and human decision is stored with timestamps and confidence scores. Complete audit trail supports GDPR right-to-explanation and government contract compliance. SQL injection prevention via parameterized queries.

Technology Decision: Local vs. Cloud

Cloud APIs (GPT-4) offer 98% raw accuracy but cost €6–10K/year and create GDPR transfer risks for government contracts. Local Llama 3.2 delivers 85% raw accuracy at zero marginal cost — boosted to 95% through architectural validation. In construction, data sovereignty + cost predictability outweigh a 3% accuracy delta.

## 📊 PORTFOLIO PROJECT: AI VALIDATION PLATFORM

### **Project Overview:**
- **Name:** AI-Powered Document Reconciliation Platform
- **Domain:** Construction contract administration
- **Use Case:** Daily Work Record (DWR) reconciliation
- **Status:** Proof-of-concept, 2-3 weeks development
- **Timeline:** Built last month (January 2026)

### **Technical Stack:**
- **Language:** Python 3.12
- **PDF Processing:** Docling + PyMuPDF (dual-library for compatibility)
- **AI Extraction:** Ollama + Llama 3.2 (local LLM, 3B parameters)
- **Validation:** Pydantic (schema validation, type checking)
- **Database:** SQLite (prototype) → PostgreSQL (production path)
- **Deployment:** Local (on-premise, data sovereignty)

### **Key Results (Quantified):**
- **Time reduction:** 85% (2-4 hours → 15-30 minutes per reconciliation)
- **Accuracy improvement:** 67% error reduction (15% manual error rate → <5% AI-assisted)
- **Extraction accuracy:** 85% raw LLM → 95% post-validation
- **Cost savings:** $55,000 annual per 10-CA team
- **ROI:** 8-month payback period
- **Zero false negatives:** Never missed a significant cost discrepancy

### **Architecture: 4-Layer Platform**

**Layer 1: Document Ingestion (Universal)**
- Handles any PDF format (structured, scanned, tables)
- Docling + PyMuPDF dual approach
- 95%+ document compatibility

**Layer 2: AI Extraction (Configurable)**
- Llama 3.2 via Ollama (local deployment)
- Schema-driven extraction (Pydantic models)
- 85% raw accuracy

**Layer 3: Validation Framework (Reusable)**
- Type validation (numbers are numbers, dates are dates)
- Range validation (0-24 hours/day, 85-100% compaction)
- Logic validation (status="failed" → compliance="false")
- Human review (5% threshold for escalation)
- Improves accuracy from 85% → 95%

**Layer 4: Storage & Audit (Universal)**
- Complete audit trail (timestamp, user, confidence, decisions)
- SQLite (prototype) → PostgreSQL (production)
- GDPR-compliant, government-ready

### **Platform Value (85-95% Code Reusability):**
- Same 4 layers work for invoice validation, contract comparison, schedule reconciliation
- Only schema + validation rules change
- Traditional approach: 8-10 weeks per new system
- Platform approach: 2-3 weeks configuration
- 70-80% time reduction per additional use case

### **Key Design Decision: Local vs Cloud LLM**
- **Cloud (GPT-4):** 88% accuracy, €125K/year, data sovereignty issues
- **Local (Llama 3.2):** 85% raw, €0 marginal cost, complete data control
- **Decision:** Local + validation framework = 95% accuracy (better than cloud raw)
- **Rationale:** Data sovereignty (government contracts), cost predictability, architectural design overcomes capability gap

### **Stakeholder Management:**
- Contract Administrators (primary users)
- Contractors (submitters)
- Ministry/Client (payers)
- Field Inspectors (verifiers)
- Finance/Accounting (processors)
- Compliance/Legal (auditors)
- IT/Engineering (infrastructure)

**Success stories:**
- Field Inspectors: Showed transparent extraction, converted skeptics to advocates
- IT: Involved early, used standard tools (Python/SQLite), addressed "shadow IT" concern
- Finance: Ran parallel validation for 1 month, AI caught 3 errors Finance missed = credibility
